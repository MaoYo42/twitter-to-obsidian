# 知识卡片批量提炼任务

请将以下帖子提炼为知识卡片，保存到对应的知识库目录。

## 输出格式

每篇输出一个 Markdown 文件，格式如下：

```markdown
# [简洁的中文标题]

> 来源：[作者] | [日期] | [原文链接]

## 核心要点

- 要点1
- 要点2
- ...

## 详细内容

[用自己的话重新组织核心内容，200-500字，不要直接复制原文]

## 个人备注

[这篇内容对我有什么用？一句话]
```

## 命名规则（⚠️ 严格执行）

文件名格式：`简洁中文标题.md`（8-25个汉字，不含特殊字符）

✅ 正确示例：
- `Claude API代理渠道选择指南.md`
- `eSIM写卡器对比评测.md`
- `Polymarket套利系统化方法论.md`

❌ 绝对禁止：
- `TW-2036047490626314247-xxx.md`（带 tweet ID）
- `@author: 开头内容截断.md`（带用户名）
- 直接移动 30_已整理 的原始文件到知识库（必须新建文件）

### 分类决策规则

1. **先看主题**，不要只看关键词。一篇关于"用 Claude Code 做 Agent"的帖子应归入 `Claude Code实战`（因为重点是 CC 操作），而不是 `AI Agent与架构`
2. **边界案例**：如果一篇帖子跨两个分类，选它**最核心的价值所在**的那个分类
3. **不确定时**：优先归入更具体的分类（如 `Claude Code实战` 优先于 `AI思维与方法论`）
4. 内容价值太低（纯营销、新闻速报无干货、段子）→ 不提炼，直接归档到 `Twitter自动化/40_已归档/`

## 写作要求

- 用中文，简洁专业
- 提炼核心知识点，不要水字数
- "个人备注"从一个 CS 本科生 + AI 工具重度用户的角度写
- 每篇 300-600 字（不含元信息）
- 读完源文件后再写，确保不遗漏关键信息
